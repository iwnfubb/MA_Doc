\chapter{Ergebnisse und Evaluation}
Das Ergebnis der fertigen Anwendung hat die Erwartungen erfüllt. Die Systemqualität wird durch eine Reihe von Verbesserung deutlich erhört. Um Feedback und Testen wird die Arbeit mit $36$ Videos durchgeführt. Die Videos haben ein Framerate von 10 Frame pro Sekunde und leisten die Auflösung von 1080x720. Um multiperspektivische Ergebnisse zu bekommen, wird die Testvideos in verschiedenen Räume gedreht. Die Testvideos wurden eine nach einander getestet, deswegen verändert sich die Bilder auch ganz stark. Auf diesem Grund wird eine Abwechselung des Testvideos auch als eine Drehung der Kamera betrachtet, weil das Hintergrundbild auch bei Drehung stark sich ändert. Im Allgemein kann das Programm außergewöhnliche Situation einer Person genau erkennen und das erfüllt das Ziel der Arbeit. Zur Visualisierung wird ein außergewöhnliche Situation mit einer gekreuzten Begrenzungsbox maskiert, wo die Testperson sich stattfindet. Die alle Testvideo wurden markiert, in welchen Zeitraum ist eine außergewöhnliche Situation passiert. Das Programm kann alle außergewöhnliche Situation erkennen, wo die Testperson auf dem Boden liegt und sich nicht mehr in 15 Bilder bewegt. Bei der Markierung wurden die außergewöhnliche Situation genau dann notiert, wenn die Testperson anfangt, auf dem Boden zu liegen. Auf diesem Grund wird das Programm nicht 100\% schaffen die Situation in jeder Zeitraum zu erkennen. Es wird hier nur die betrachtet, ob das System die außergewöhnliche in einem Zeitraum richtig detektieren kann. Zum Beispiel wird eine abnormale Situation von Frame 100 bis 200 notiert und unseres System sagt von Frame 120 bis 210. Dann das ist nicht 100\% richtig, aber die Erkennung von der abnormale Situation von Frame 100 bis 200 ist erfolgreich.\\

Für Messung der Laufzeit wurden die Tests auf einem Rechner Intel-Core i5 2.60 GHz ausgeführt. Die Laufzeit wird mehr mal gemessen und für jedes Bild braucht das System ein durchschnittlicher Wert von 109 Millisekunde die Verarbeitungszeit. Das bedeutet in eine Sekunde kann das System 100 Bilder mit Auflösung von 1280x720 schaffen. Damit wird auch unseres zweite Ziel realisiert und das Programm ist eine Echtzeitanwendung zur Erkennung der außergewöhnlichen Situation.
